## Finding Q: IxMonad do-elaboration の Lift 制限

`elaborate_do_monadic.go` の `checkDo` は IxMonad do-elaboration で
Lift-based 解決を意図的に除外している。

**根拠**: `Lift m r1 r2 a = m a` は行パラメータを消去するため、
`monadHead` に部分適用された行が含まれる場合（例: `State pre post`）、
`Lift (State pre post)` は pre/post の threading を失い、
pre ≠ post のとき elaboration が型的に不正になる。

**現在の防御**:

1. `hasMonadInstance` → `Type → Type` モナドは AST レベルで `mbind`/`mpure` に desugar
2. `hasDirectIxMonadInstance` → `Row → Row → Type → Type` 型は直接 IxMonad 解決のみ
3. 上記いずれにも該当しない → コンパイルエラー

`extractIxMethod` の Lift fallback は do-block 外の `pure` elaboration
（`bidir_cbpv.go`）で必要。`Type → Type` モナドに対しては Lift は安全。
